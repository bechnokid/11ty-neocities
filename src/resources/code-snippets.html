"Old-Fashioned" Marquee with Pure CSS

I remembered how marquees used to move in a more "choppy" manner before they officially became "deprecated". If you want your sites to really have the Y2K aesthetic, choppy marquees are the way to go, in my opinion!

CSS

{% details { title: "(Click to show/hide code)", summaryCls: "details h4" } %} {% renderTemplate "md" %} ```css/ @keyframes marquee { 0% { transform: translateX(100%); } 100% { transform: translateX(-150%); } } .marquee-container { overflow: hidden; white-space: nowrap; width: 100%; } .marquee-content { display: inline-block; animation: marquee 30s steps(400, end) infinite; } .marquee-content > * { margin: 0 .25rem; } .marquee-content img { vertical-align: middle; } ``` {% endrenderTemplate %} {% enddetails %}

Notes

{% renderTemplate "md" %} - **Lines 2-3**: the `translateX` values are to ensure that all items in the example "disappear" before cycling through again. These numbers can change depending on how wide the marquee will be and how many items will be in it. Feel free to experiment! - For the marquee to scroll from left to right (as opposed to right to left in the example), reverse the numbers. (change 100% to -150%, and change -150% to 100%) - **Line 9**: this width can be any number. I just chose 100% so that it would span the width of this page. - **Line 14**: `30s` can be changed to have the following effects: - Lower number = faster scrolling - Higher number = slower scrolling - **Line 14**: 400 in `steps()` can be changed to have the following effects: - Lower number = choppier movement - Higher number = smoother movement - **Lines 17-19**: these lines are only here to ensure there is some spacing between the items within the container. This is optional and can be removed if you'd like. {% endrenderTemplate %}
